Change Logs
===========

Purpose
- Use this file to record version changes for the project and key dependencies.
- Add one entry whenever a version is added, upgraded, downgraded, or removed.

Recommended Format
- Date: YYYY-MM-DD
- Area: Backend | Frontend | Testing | CI/CD | Infrastructure | Document
- Item: package, framework, SDK, runtime, image, or tool name
- From: previous version
- To: new version
- Reason: short explanation
- Updated By: person or AI/automation name
